# GitHub Copilot Instructions for Next.js Project

This is a modern Next.js 15 application with the following:

## Architecture

- **Framework**: Next.js 15 (App Router)
- **Styling**: Tailwind CSS v4 with monochrome white theme
- **Components**: shadcn/ui library
- **Language**: TypeScript
- **Theme**: Supports light/dark modes with localStorage persistence

## Project Structure

- `/src/app` - Next.js pages and routes
- `/src/components` - React components (UI + common)
- `/src/hooks` - Custom React hooks
- `/src/lib` - Utilities and helpers
- `/src/types` - TypeScript definitions
- `/src/utils` - Pure utility functions
- `/src/config` - Application config

## Key Files

- `src/app/layout.tsx` - Root layout with Header/Footer
- `src/app/page.tsx` - Home page
- `src/app/dashboard/` - Dashboard example
- `src/app/api/route.ts` - API example
- `src/components/common/Header.tsx` - Header with theme toggle
- `src/hooks/useTheme.ts` - Theme management hook
- `src/app/globals.css` - Theme colors and global styles

## Theme

The application uses a monochrome white theme defined in `globals.css`:
- Light mode: White background, black text
- Dark mode: Black background, white text
- All colors are grayscale using oklch() color model

## Commands

- `npm run dev` - Start dev server (port 3000)
- `npm run build` - Production build
- `npm start` - Run production build
- `npm run lint` - Run ESLint

## Adding Components

Add shadcn/ui components with:
```bash
npx shadcn add [component-name]
```

## Code Generation Tips

- Use the Button component as a reference for styling
- Follow the established folder structure
- Use TypeScript for type safety
- Extend from existing components/hooks when possible
- Keep components in `/components/ui` for shadcn components
- Keep custom components in `/components/common`
